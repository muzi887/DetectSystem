# 第一章 系统简介

## 1.1 系统背景与定位

青禾智匠——作物灾害智慧监测预警系统（以下简称“本系统”）是在人工智能与现代农业深度融合背景下，面向农业生产灾害监测与预警需求的智慧化软件平台。系统以“空天地一体化”数据采集架构为基础，融合卫星遥感、无人机多光谱、物联网传感器与气象站等多源异构数据，通过 AI+GIS 双引擎技术，实现对作物生长状况、病虫害、气候灾害的智能识别、实时监测与精准预警，最终达到降低灾害损失、保障粮食安全的目标。

## 1.2 核心价值

- **空天地一体化**：整合卫星（天）、无人机（空）、地面传感器（地）与 GIS 地理信息数据，形成全方位监测网络。  
- **AI+GIS 双引擎**：基于深度学习的病害识别与 GIS 空间分析，提供从“发现异常”到“定位处置”的闭环决策支撑。  
- **降低灾害损失**：通过提前预警与智能预案建议，帮助农户与管理部门提前采取防灾减灾措施，最大限度减少因病虫害、旱涝、极端天气等造成的经济损失。

## 1.3 系统功能概览

本系统包含六大核心模块：**相关数据模块**、**灾害实时监测模块**、**智能分析模块**、**灾害预警模块**、**智慧决策模块**及**用户与系统管理**。各模块相互协同，为用户提供从数据查询、态势感知、智能诊断到预警处置与决策支持的全流程服务。

## 1.4访问方式与登录

1. **打开系统**：建议使用 Google Chrome、Microsoft Edge 等主流浏览器，在地址栏输入并访问网址：`http://82.157.234.123:88`。
2. **输入凭证**：在登录界面输入系统分配的账号（如：13800000000）与密码（如：123456），勾选用户协议后点击“立即登录”即可进入系统首页。

### 2.1.2 账号密码输入

**操作步骤：**

1. **点击账号输入框** → 光标自动定位至账号输入区域。

- 登录前请确认网络连接正常。  
- 若多次登录失败，请核对账号与密码，或联系管理员重置。  
- 登录成功后，系统将保持会话状态，直至用户主动退出或会话超时。

## 4.1 相关数据模块

相关数据模块整合卫星、无人机、传感器及 GIS 等多源数据，提供空天地数据检索、传感器数据详情查看、自动生成数据简报及 GIS 热力图、时序折线图展示等功能。

### 4.1.1 空天地数据检索与查看

**进入方式：** 点击导航栏中的“相关数据” → 进入相关数据模块主界面。

**界面布局：** 页面分为左侧主内容区与右侧导航按钮区。右侧提供四个数据源 Tab：传感器数据（地）、无人机遥感（空）、气象数据（天）、GIS 数据（图）。点击不同 Tab 可切换至对应数据视图。

**操作步骤：**

1. **点击“传感器数据(地)”** → 主内容区显示物联网传感器监控视图，包括“最近 7 天环境参数趋势”的时序折线图。  
2. **点击“无人机遥感(空)”** → 主内容区切换为无人机多光谱监测视图，展示 NDVI 植被指数分析热力图。  
3. **点击“气象数据(天)”** → 主内容区切换为气象站实时数据，以卡片形式展示实时温度、相对湿度、降水概率。  
4. **点击“GIS 数据(图)”** → 主内容区切换为地理信息可视化，展示土壤墒情热力分布图。

**预期效果：** 每次切换 Tab 时，页面会短暂显示“正在接入异构数据流...”的加载提示，加载完成后展示对应数据源的可视化内容。

![[attachments/4.1.1.png]]

### 4.1.2 传感器微环境数据可视化（时序图/热力图操作）

**传感器数据（地）——时序折线图：**

1. **确保当前 Tab 为“传感器数据(地)”** → 主内容区顶部显示标题“物联网传感器监控”及副标题“最近 7 天环境参数趋势”。  
2. **查看折线图** → 折线图横轴为日期（如 11/20、11/21…），纵轴为异常预警次数，曲线反映过去 7 天各日的预警数量变化趋势。  
3. **鼠标悬停** → 在折线图上悬停可显示对应日期的详细 Tooltip，如“11/20 报警数量: 3 次”。  
4. **查看 AI 智能分析** → 主内容区下方设有“AI 智能分析”区域，系统根据当前传感器数据自动生成简要结论，如“过去 7 天传感器网络运行平稳”或“检测到 X 次高风险异常，建议立即派人排查”。

**无人机遥感 / GIS 数据——热力图：**

1. **切换至“无人机遥感(空)”或“GIS 数据(图)”** → 主内容区展示热力图模拟效果。  
2. **查看热力图** → 热力图展示 NDVI 植被指数或土壤湿度空间分布，不同颜色代表不同强度等级。  
3. **查看叠加信息** → 热力图上方或叠加区域会显示数据来源说明（如 DJI-Mavic-3M、Sentinel-2 Satellite）。

**气象数据——仪表盘：**

1. **切换至“气象数据(天)”** → 主内容区显示三个气象卡片。  
2. **查看实时温度** → 第一个卡片显示当前温度（如 32°C）。  
3. **查看相对湿度** → 第二个卡片显示相对湿度（如 65%）。  
4. **查看降水概率** → 第三个卡片显示降水概率（如 15%）。

## 4.2 灾害实时监测模块

灾害实时监测模块以 GIS 地图为核心，展示多源数据（卫星、无人机、传感器）融合的监测点分布与状态，支持点位详情查看、手动触发预警及标记解决等操作。

### 4.2.1 GIS 地图监测点分布查看

**进入方式：** 点击导航栏中的“灾害实时监测” → 进入地图监测界面。

**界面布局：** 页面包含一个主地图卡片（标题为“地图 - 监测点实时分布”）及一个操作面板卡片（标题为“地图操作”）。地图采用深色底图，监测点以聚类形式展示，不同颜色代表不同状态：绿色表示正常（normal），橙色表示警告（warning），红色表示危急（critical）。

**操作步骤：**

1. **进入页面** → 系统自动加载地图底图并请求监测点数据。  
2. **查看监测点分布** → 地图上显示各监测点标记，近距离时显示为独立点位，远距离时自动聚合成簇，簇上显示包含的点位数量。  
3. **点击“缩放至全部”** → 位于“地图操作”卡片内，点击后地图自动调整视野，使所有监测点均 visible 在可视范围内。  
4. **点击“刷新数据”** → 系统重新请求监测点与预警数据，刷新后地图上的点位与状态同步更新。

**预期效果：** 用户可快速掌握所有监测点的空间分布及整体状态，便于发现异常区域。

![[attachments/4.2.1.png]]

### 4.2.2 监测点状态与数据融合面板说明

**查看单个监测点详情：**

1. **点击地图上的监测点标记** → 弹出该点的详情弹窗（Popup）。  
2. **查看弹窗内容** → 弹窗内显示：监测点名称、温度、土壤湿度、状态（normal/warning/critical）。若该点存在未处理预警，还会显示“未处理预警: XXX”的红色提示。  
3. **手动触发预警** → 点击弹窗内的“手动触发”按钮，系统将自动创建一条中等级别的手动触发预警，内容为“手动触发：XXX 状态异常”，并更新弹窗中的预警信息。  
4. **标记解决** → 若该点有未处理预警，可点击“标记解决”按钮，将该条预警标记为已处理，弹窗中的未处理预警提示将消失。

**预期效果：** 用户可在不离开地图的情况下，快速了解单点环境数据、触发新预警或关闭已完成预警。

## 4.3 智能分析模块

智能分析模块包括“作物生长状况评估”与“灾害识别与分类”两大功能，支持用户上传作物图片，基于深度学习模型进行长势/病害识别，并自动创建预警记录。

### 4.3.1 作物生长状况评估（上传图片、深度学习分析、查看指标）

**进入方式：** 点击导航栏中的“智能分析” → 进入智能分析界面。

**界面布局：** 主内容区包含一个卡片，标题为“智能分析”。卡片内分为左侧表单区和右侧分类区。左侧为图片上传区、作物类型选择、补充信息输入及“确定”按钮；右侧为四个分类按钮：灾害识别、病虫害识别、气候灾害识别、其他。

**操作步骤：**

1. **点击上传区域** → 上传区域为 250×250 像素的图片卡片，显示“上传图片”及加号图标。  
2. **选择本地图片** → 从本地选择一张 JPG 或 PNG 格式的作物图片（大小不超过 2MB）。  
3. **等待上传** → 上传过程中显示进度动画，上传成功后图片将预览在卡片内。  
4. **选择作物类型** → 在左侧下拉框中选择作物类型：桃、苹果、小麦、水稻。  
5. **填写补充信息**（可选） → 在“其他补充信息”文本框中输入与本次分析相关的说明。  
6. **选择分析分类** → 在右侧点击“灾害识别”“病虫害识别”“气候灾害识别”或“其他”，被选中的按钮将高亮。  
7. **点击“确定”或“识别”** → 系统调用后端 AI 接口进行分析，分析完成后自动创建一条预警记录，并提示“分析完成！结果已自动存入预警列表。”

**预期效果：** 用户获得基于深度学习的识别结果（如“桃疮痂病”），系统根据识别结果自动生成预警并写入灾害预警模块，用户可在“灾害预警”页面查看。

![[attachments/4.3.1.png]]

### 4.3.2 灾害识别与分类（选择病虫害/气候灾害、AI 识别操作、查看严重程度）

**灾害类型对应关系：**

- **灾害识别**：适用于广义的农业灾害，如病害、缺素等。  
- **病虫害识别**：适用于昆虫病害、细菌真菌类病害。  
- **气候灾害识别**：适用于冻害、热害、干旱、渍涝等气候相关灾害。  
- **其他**：适用于上述分类之外的情形。

**操作步骤：**

1. **上传图片** → 按要求上传清晰的作物或病害部位照片。  
2. **选择分类** → 根据实际场景点击“病虫害识别”或“气候灾害识别”等。  
3. **点击“确定”** → 系统执行 AI 识别，并将识别结果与置信度（如 92%）写入预警消息。  
4. **查看严重程度** → 若识别结果为非健康状态，系统将自动创建高级别（high）预警；若为健康，则创建低级别（low）预警。用户可在“灾害预警”模块的预警列表中查看级别标签（低/中/高）及完整消息内容。

## 4.4 灾害预警模块

灾害预警模块支持预警阈值设定（通过新建预警时选择级别实现）、AI 自动触发与手动触发预警、预警状态标记（待处理/已处理）及预警信息发布与指挥调度。

### 4.4.1 预警阈值设定与规则管理

本版本中，预警的“阈值”主要通过新建预警时选择的“预警级别”来体现。用户可根据业务规则，将不同严重程度的异常对应为低、中、高三级预警。

**操作步骤：**

1. **进入灾害预警页面** → 点击导航栏中的“灾害预警”。  
2. **点击“新建预警”** → 位于卡片右上角。  
3. **在弹窗中选择监测点** → 从下拉列表中选择目标监测点，支持搜索筛选。  
4. **选择预警级别** → 从“低”“中”“高”中选择其一，对应不同的处置优先级。  
5. **输入预警信息** → 在“预警信息”文本框中输入详细的预警内容描述。  
6. **点击“确定”** → 完成新建，预警将出现在列表中。

**预期效果：** 新建的预警以列表形式展示，带有级别标签（低/中/高）及待处理状态。

![[attachments/4.4.1.png]]

### 4.4.2 预警信息处理流程（待处理到已处理的状态标记）

**查看预警列表：**

预警列表以卡片形式展示，每条预警包含：级别标签、监测点名称、时间、预警内容、状态标签（待处理/已处理）及操作链接。

**操作步骤：**

1. **查看待处理预警** → 状态为“待处理”的预警以红色标签显示，需优先处置。  
2. **点击“标记解决”** → 对某条待处理预警，点击其右侧的“标记解决”链接，系统将该条预警状态更新为“已处理”，标签变为绿色。  
3. **点击“标记未处理”** → 对已处理的预警，可点击“标记未处理”将其恢复为待处理状态，便于重新跟踪。  
4. **点击“删除”** → 可删除不再需要的预警记录（删除操作需谨慎，删除后不可恢复）。

**预期效果：** 预警状态在“待处理”与“已处理”之间灵活切换，便于管理人员跟进处置进度。

![[attachments/4.4.2.png]]

### 4.4.3 预警信息发布与指挥调度

**新建预警**即视为一次预警“发布”行为，发布后该预警会：

- 出现在预警列表中，供所有有权限的用户查看；  
- 在“灾害实时监测”地图上，对应监测点的弹窗中显示“未处理预警”提示；  
- 在“智慧决策”模块的“待处理灾害预警”列表中展示，用于后续决策与调度。

**指挥调度流程建议：**

1. 在灾害预警页面新建或查看预警；  
2. 在灾害实时监测地图上定位到对应监测点，查看现场数据；  
3. 在智慧决策模块中选择该预警，获取 AI 决策建议与地理信息；  
4. 根据建议执行处置后，回到灾害预警页面将该预警“标记解决”。

## 4.5 智慧决策模块

智慧决策模块结合 AI 与 GIS，提供地形农田分布等地理信息，以及基于预警内容与监测数据的降温、灌溉等防灾减灾预案和智能决策建议。

### 4.5.1 地理信息与资源环境查看

**进入方式：** 点击导航栏中的“智慧决策” → 进入智慧决策支持界面。

**界面布局：** 主内容区分为左右两栏。左侧为“待处理灾害预警”列表及“区域概览”卡片（含小地图与经纬度信息）；右侧为“实时监测数据”与“AI 决策建议”两个卡片。未选择预警时，右侧显示“请从左侧列表选择一个预警进行决策分析”的占位提示。

**操作步骤：**

1. **查看待处理预警列表** → 左侧列表展示所有状态为“待处理”的预警，每条显示监测点名称、级别标签及预警内容摘要。  
2. **点击某条预警** → 该条预警高亮，右侧“区域概览”卡片显示对应监测点的小地图及经纬度。  
3. **查看地理信息** → 小地图中标出该监测点位置，下方描述项显示经度、纬度等地理坐标。

**预期效果：** 用户可快速定位预警对应的地理位置，为现场勘查与资源调配提供依据。

### 4.5.2 智能决策模型与预案生成（降温/灌溉/打药等建议）

**操作步骤：**

1. **从左侧选择一条待处理预警** → 点击列表中的某条预警条目。  
2. **查看实时监测数据** → 右侧“实时监测数据”卡片展示该点位的设备状态、当前温度、土壤湿度等。  
3. **查看 AI 决策建议** → “AI 决策建议”卡片中列出系统根据预警级别、内容及监测数据自动生成的处置建议，例如：  
   - “最高优先级处理！立即通知所有相关应急负责人。”  
   - “目标区域土壤湿度为 X%，建议立即启动远程灌溉系统。”  
   - “目标区域温度已达 X°C，建议启动田间降温预案（如喷雾）。”  
   - “派遣运维人员前往现场检修硬件设备。”  
4. **导出方案** → 点击“AI 决策建议”卡片右上角的“导出方案”按钮，可将当前决策建议导出为文件，便于下发与归档。

**预期效果：** 用户获得针对具体预警的、可执行的防灾减灾预案，缩短从发现异常到采取行动的时间。

# 第五章 常见问题与注意事项

## 5.1 登录与访问

- **问：忘记密码怎么办？** 
- 答：登录界面提供“忘记密码”链接，可尝试通过该流程找回；若不可用，请联系系统管理员重置密码。

- **问：登录后页面提示 404、500 错误，或一直提示账号密码错误？** 
- 答：
  1. 请检查输入的账号密码是否包含多余空格（测试账号：13800000000 / 123456）。
  2. 若页面白屏或 404，可直接刷新页面（系统已配置 Nginx 路由重定向恢复）。
  3. 若提示服务器内部错误（500），请联系运维管理员检查后台 PM2 (Node.js) 或 Flask (Python) 服务进程是否因端口占用而异常断开。

## 5.2 数据与地图

- **问：地图上监测点不显示？**  
  答：请检查网络连接，并点击“刷新数据”按钮；若仍无数据，可能是当前账户下暂无监测点或后端服务异常。

- **问：相关数据模块切换 Tab 后加载很久？**  
  答：系统会模拟异构数据接入延迟，通常 500ms 内完成；若超过数秒仍无响应，请检查网络或联系技术支持。

## 5.3 智能分析

- **问：上传图片后分析失败？**  
  答：请确认图片格式为 JPG 或 PNG，且大小不超过 2MB；同时需确保智能分析后端服务（如 Python Flask 服务）已启动且地址配置正确。

- **问：分析结果不准确？**  
  答：AI 识别结果受图片质量、拍摄角度、光照等影响，建议上传清晰、主体明确的照片；识别结果仅供参考，重要决策请结合人工判断。

## 5.4 预警与决策

- **问：新建预警后在地图上找不到？**  
  答：新建预警会绑定到所选监测点，请在地图上找到对应监测点并点击查看弹窗；若该点已有其他未处理预警，新预警会一并显示。

- **问：智慧决策中没有建议？**  
  答：请先从左侧选择一条“待处理”预警；若列表为空，请先在灾害预警模块新建或确保存在未处理的预警记录。

## 5.5 安全与使用规范

- 请勿在公共计算机上勾选“记住登录”类选项，使用完毕后务必点击“退出登录”。  
- 预警信息可能涉及敏感业务，请按单位规定做好权限管理与数据保密。  
- 系统版本更新后，部分界面与操作可能有所调整，请以实际系统为准。
