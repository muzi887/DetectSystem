# 源程序导出至 Word 文档 - 实现说明

本文档说明如何将作物灾害预警系统的源程序按规范整理并追加到 Word 文档中。

---

## 一、需求概述

根据申报材料要求，源程序需整理到 Word 文档中，并满足以下规则：

| 规则 | 说明 |
|------|------|
| 模块分离 | 各模块分开提交 |
| 前 30 页 | 每个模块提交前 30 页连续代码 |
| 后 30 页 | 每个模块提交后 30 页连续代码 |
| 每页行数 | 每页 ≥ 50 行（不含空行） |
| 不足 3000 行 | 若模块总行数 < 3000，则全部提交 |
| 最后一页 | 最后一页应是某段程序的结束页 |
| 代码较多时 | 尽量多提供，便于后期排版 |

---

## 三、核心逻辑

### 3.1 行数统计

- **有效行**：仅统计非空行（`line.strip()` 不为空）
- **每页行数**：50 行
- **阈值**：3000 行（不足则全部提交）

```python
# scripts/add_code_to_docx.py
LINES_PER_PAGE = 50
PAGES_FRONT = 30
PAGES_BACK = 30
LINES_FRONT = PAGES_FRONT * LINES_PER_PAGE  # 1500
LINES_BACK = PAGES_BACK * LINES_PER_PAGE    # 1500
TOTAL_THRESHOLD = 3000  # 不足3000行全部提交

def get_non_empty_lines(content: str) -> list[str]:
    """获取非空行列表"""
    return [line for line in content.splitlines() if line.strip()]
```

### 3.2 代码提取规则

```
若 模块总有效行数 < 3000：
    全部提交，按文件顺序拼接

若 模块总有效行数 ≥ 3000：
    提交前 1500 行（30 页 × 50 行）
    + 中间省略说明
    + 后 1500 行（30 页 × 50 行）
```

**最后一页处理**：后 1500 行从 `总行数 - 1500` 开始取到末尾，自然结束于最后一个文件的最后一行，满足「最后一页是程序结束页」的要求。

核心实现如下：

```python
def extract_code_for_module(files_data, total_non_empty):
    if total_non_empty < TOTAL_THRESHOLD:
        # 全部提交
        result = []
        for filepath, content in files_data:
            result.append(f"\n// ========== {filepath} ==========\n")
            result.append(content)
        return "".join(result)

    # 前30页 (1500行) + 后30页 (1500行)
    lines_collected = []
    for filepath, content in files_data:
        non_empty = get_non_empty_lines(content)
        for line in non_empty:
            lines_collected.append((filepath, line))

    # 前1500行
    front_lines = [...]
    # 后1500行 - 从 total - 1500 开始，自然结束于最后一行
    start_back = max(LINES_FRONT, len(lines_collected) - LINES_BACK)
    back_lines = [...]

    return "".join(front_lines) + "\n\n// ... (中间部分省略) ...\n\n" + "".join(back_lines)
```

### 3.3 文件边界标识

每个文件开头插入分隔注释，便于阅读：

```

```

## 四、防重复机制

为避免多次运行脚本导致源程序清单重复追加，实现了「先删后加」逻辑：

1. **查找**：遍历 Word 文档 body 中的元素，提取文本内容
2. **定位**：找到第一个包含「源程序清单」的段落
3. **删除**：删除该段落及之后的所有内容（段落、表格等）
4. **追加**：再添加新的源程序清单内容

实现代码如下：

```python
def _get_element_text(elem) -> str:
    """从 OXML 元素中提取文本"""
    texts = []
    for t in elem.iter():
        tag = t.tag.split("}")[-1] if "}" in str(t.tag) else str(t.tag)
        if tag == "t":
            if t.text:
                texts.append(t.text)
    return "".join(texts)

def remove_existing_source_section(doc: Document) -> None:
    """删除文档中已有的「源程序清单」及之后的所有内容，避免重复添加"""
    body = doc.element.body
    to_remove = []
    found = False
    for child in body:
        if found:
            to_remove.append(child)
            continue
        text = _get_element_text(child)
        if "源程序清单" in text:
            found = True
            to_remove.append(child)
    for elem in to_remove:
        body.remove(elem)
```

## 五、Word 文档格式

| 项目 | 设置 |
|------|------|
| 代码字体 | Consolas 9pt |
| 中文 | 宋体 |
| 分块 | 每块约 50 行，便于排版 |
| 模块标题 | 一级标题，居中 |
| 说明文字 | 每模块前显示「本模块共 X 行有效代码（不含空行）」 |

写入 Word 时的格式设置：

```python
# 分块添加，每块约 50 行
chunk_size = LINES_PER_PAGE  # 50
chunks = [lines[i:i + chunk_size] for i in range(0, len(lines), chunk_size)]

for chunk in chunks:
    p = doc.add_paragraph()
    run = p.add_run("\n".join(chunk))
    run.font.name = "Consolas"
    run.font.size = Pt(9)
    r = run._element
    if r.rPr is None:
        r.get_or_add_rPr()
    r.rPr.rFonts.set(qn("w:eastAsia"), "宋体")  # 中文字体
```

- 安装 Python 3
- 安装依赖：`pip install python-docx`
- 确保 `docs/作物灾害预警系统使用说明.docx` 存在

2. **运行脚本**：

3. **查看结果**：打开 `docs/作物灾害预警系统使用说明.docx`，文档末尾应包含「源程序清单」及各模块代码

- **文件不存在**：脚本会提示「找不到文档」并退出

## 七、脚本结构概览

```
add_code_to_docx.py
├── 常量配置（路径、行数、阈值）
├── get_non_empty_lines()         # 获取非空行
├── extract_code_for_module()     # 按规则提取代码
├── remove_existing_source_section()  # 删除已有源程序部分
├── _get_element_text()           # 从 OXML 提取文本
├── MODULES                       # 模块与文件映射
└── main()                        # 主流程
```

主流程 `main()` 核心逻辑：

```python
def main():
    doc = Document(str(DOC_PATH))

    # 1. 先删除已有的源程序清单部分，避免重复
    remove_existing_source_section(doc)

    # 2. 添加分页和总览标题
    doc.add_page_break()
    doc.add_heading("源程序清单", level=1)

    # 3. 遍历各模块，读取文件、提取代码、分块写入
    for module_name, file_paths in MODULES:
        files_data = []
        for rel_path in file_paths:
            full_path = PROJECT_ROOT / rel_path
            content = open(full_path, encoding="utf-8").read()
            files_data.append((rel_path, content))
        total_non_empty = sum(len(get_non_empty_lines(c)) for _, c in files_data)
        code = extract_code_for_module(files_data, total_non_empty)
        lines = code.split("\n")
        chunks = [lines[i:i+50] for i in range(0, len(lines), 50)]
        for chunk in chunks:
            doc.add_paragraph("\n".join(chunk))  # 字体等格式见上文

    doc.save(str(OUTPUT_PATH))
```
