# 源程序导出至 Word 文档 - 实现说明

本文档说明如何将作物灾害预警系统的源程序按规范整理并追加到 Word 文档中。

---

## 一、需求概述

根据申报材料要求，源程序需整理到 Word 文档中，并满足以下规则：

| 规则 | 说明 |
|------|------|
| 模块分离 | 各模块分开提交 |
| 前 30 页 | 每个模块提交前 30 页连续代码 |
| 后 30 页 | 每个模块提交后 30 页连续代码 |
| 每页行数 | 每页 ≥ 50 行（不含空行） |
| 不足 3000 行 | 若模块总行数 < 3000，则全部提交 |
| 最后一页 | 最后一页应是某段程序的结束页 |
| 代码较多时 | 尽量多提供，便于后期排版 |

---

## 二、实现方案

### 2.1 技术选型

- **语言**：Python 3
- **依赖**：`python-docx`（读写 Word 文档）
- **脚本位置**：`scripts/add_code_to_docx.py`

### 2.2 模块划分

将项目源程序划分为 5 个逻辑模块：

| 模块 | 包含文件 |
|------|----------|
| 模块一：入口与配置 | `main.ts`, `App.vue`, `vite.config.ts`, `router/index.ts` |
| 模块二：布局组件 | `AppLayout.vue` |
| 模块三：视图组件 | `About.vue`, `Home.vue`, `Login.vue`, `DataAnalysis.vue`, `MapVisualization.vue`, `RelatedData.vue`, `WarningSystem.vue`, `DecisionSupport.vue` |
| 模块四：状态管理与接口 | `data.ts`, `user.ts`, `http.ts`, `analysis.ts`, `useGlobalSearch.ts` |
| 模块五：Mock 与后端服务 | `server.ts`, `app.py` |

---

## 三、核心逻辑

### 3.1 行数统计

- **有效行**：仅统计非空行（`line.strip()` 不为空）
- **每页行数**：50 行
- **阈值**：3000 行（不足则全部提交）

### 3.2 代码提取规则

```
若 模块总有效行数 < 3000：
    全部提交，按文件顺序拼接

若 模块总有效行数 ≥ 3000：
    提交前 1500 行（30 页 × 50 行）
    + 中间省略说明
    + 后 1500 行（30 页 × 50 行）
```

**最后一页处理**：后 1500 行从 `总行数 - 1500` 开始取到末尾，自然结束于最后一个文件的最后一行，满足「最后一页是程序结束页」的要求。

### 3.3 文件边界标识

每个文件开头插入分隔注释，便于阅读：

```
// ========== src/views/user/About.vue ==========
```

## 四、防重复机制

为避免多次运行脚本导致源程序清单重复追加，实现了「先删后加」逻辑：

1. **查找**：遍历 Word 文档 body 中的元素，提取文本内容
2. **定位**：找到第一个包含「源程序清单」的段落
3. **删除**：删除该段落及之后的所有内容（段落、表格等）
4. **追加**：再添加新的源程序清单内容

实现函数：`remove_existing_source_section(doc)`

## 六、使用方式

### 6.1 前置条件

- 安装 Python 3
- 安装依赖：`pip install python-docx`
- 确保 `docs/作物灾害预警系统使用说明.docx` 存在

### 6.2 执行步骤

1. **关闭 Word**：若文档正在 Word 中打开，请先关闭
2. **运行脚本**：
   ```bash
   cd d:\code2\software\vue\program\DetectSystem
   python scripts/add_code_to_docx.py
   ```
3. **查看结果**：打开 `docs/作物灾害预警系统使用说明.docx`，文档末尾应包含「源程序清单」及各模块代码

### 6.3 异常处理

- **文件被占用**：若原文档被 Word 打开，脚本会尝试保存到 `作物灾害预警系统使用说明_含源码.docx`
- **文件不存在**：脚本会提示「找不到文档」并退出

## 七、脚本结构概览

```
add_code_to_docx.py
├── 常量配置（路径、行数、阈值）
├── get_non_empty_lines()         # 获取非空行
├── extract_code_for_module()     # 按规则提取代码
├── remove_existing_source_section()  # 删除已有源程序部分
├── _get_element_text()           # 从 OXML 提取文本
├── MODULES                       # 模块与文件映射
└── main()                        # 主流程
```
