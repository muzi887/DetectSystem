# DetectSystem 模拟数据说明

本文档解释本项目中**模拟数据（Mock Data）**的来源、用途、工作方式及如何运行。

---

## 一、模拟数据概览

项目中的模拟数据分为**三个部分**：

| 类型 | 位置 | 用途 |
|------|------|------|
| JSON Server 模拟后端 | `src/mock/` | 用户、监测点、预警等 REST API |
| Flask AI 模拟 | `server/app.py` | 病害识别分析接口的假数据 |
| 前端内联模拟 | 各 Vue 组件 | 图表、热力图、结论文案等占位数据 |

---

## 二、JSON Server 模拟后端

### 2.1 文件与结构

- **入口**：`src/mock/server.ts`
- **数据源**：`src/mock/db.json`

`db.json` 中定义了以下数据表（资源）：

#### users（用户）

用于登录校验。示例账号：

| 手机号 | 密码 | 角色 |
|--------|------|------|
| 13800000000 | 123456 | user |
| 13911112222 | 123 | user |
| 18800001111 | admin | admin |

#### monitorPoints（监测点）

包含 `id`、`name`、`lat`/`lng`、`temp`、`soilMoisture`、`status` 等字段，用于地图展示和监测站管理。

#### alerts（预警）

包含 `id`、`pointId`、`level`、`message`、`time`、`handled` 等字段，用于预警列表和统计。

### 2.2 工作方式

1. 使用 **json-server** 基于 `db.json` 自动生成 REST 接口：
   - `GET /monitorPoints` 获取监测点列表
   - `GET /alerts` 获取预警列表
   - `POST /alerts` 新建预警
   - `DELETE /alerts/:id` 删除预警

2. **自定义 `/login` 接口**：`server.ts` 中手动实现，通过手机号 + 密码在 `db.users` 中查找，成功则返回 `token` 和 `user` 信息。

3. **数据持久化**：json-server 会将 PUT/POST/DELETE 的变更写回 `db.json`。

### 2.3 如何启动

```bash
npm run mock
```

默认端口：**3000**（可通过环境变量 `MOCK_PORT` 修改）。

## 三、Vite 代理与请求路由

前端统一以 `/api` 为前缀发请求，由 `vite.config.ts` 的 `proxy` 进行转发：

```
前端请求                    代理目标                    说明
───────────────────────────────────────────────────────────────
/api/analysis/*      →  http://127.0.0.1:5000     Flask 智能分析
/api/*               →  http://localhost:3000      json-server（会去掉 /api 前缀）
```

- `/api/analysis/image` → 转发到 Flask 5000 端口
- `/api/login`、`/api/monitorPoints`、`/api/alerts` 等 → 转发到 json-server，并去掉 `api` 前缀

因此，开发时需要**同时**运行：
1. `npm run mock` —— JSON Server（3000）
2. `npm run dev` —— Vite 开发服务器（5173）
3. （若使用 AI 分析）Python Flask 服务（5000）

## 四、Flask AI 模拟（病害识别）

### 4.1 文件位置

`server/app.py`

### 4.2 模拟逻辑

当尚未接入真实 AI 模型时，使用 `MOCK_RESULTS` 字典模拟病害识别结果：

```python
MOCK_RESULTS = {
    "peach": ["桃疮痂病", "桃褐腐病", "桃缩叶病", "健康"],
    "apple": ["苹果腐烂病", "苹果轮纹病", "健康"],
    "wheat": ["小麦锈病", "小麦赤霉病"],
    "rice": ["稻瘟病", "纹枯病"]
}
```

- 根据前端传入的 `cropType`（作物类型）选择对应病害列表
- 用 `random.choice()` 随机返回一个病害名称
- 置信度在 0.85–0.99 之间随机
- 模拟 1 秒延迟（`time.sleep(1)`）

### 4.3 接入真实模型

在 `run_ai_model_prediction()` 中替换当前模拟逻辑，改为调用实际的图像识别模型即可。

## 五、前端内联模拟

部分页面在缺少真实数据时，直接在前端使用“占位”逻辑或样式。

### 5.1 RelatedData.vue

- **无人机 / GIS 热力图**：用 `.heatmap-mock` 的 CSS 模拟热力图效果，非真实地图或遥感数据
- **其他 Tab 的 AI 结论**：`otherConclusions` 对象中写死文案（如“区域 A3 出现轻微缺氮……”），仅用于展示

### 5.2 DataAnalysis.vue

- **上传进度**：`setTimeout` 模拟上传过程，让用户看到 loading 效果
- 若未实际请求后端，可在控制台看到“前端模拟上传成功，未发送网络请求”等日志

### 5.3 data.ts（Pinia Store）

- **坐标校准**：对 `lat`/`lng` 增加 `0.00001` 偏移，模拟纠偏算法
- 实际项目中应改为真实的坐标转换（如 proj4 等）

## 六、数据流示意

```
┌─────────────────────────────────────────────────────────────────────┐
│                        前端 Vue 应用 (port 5173)                      │
│  http.get('/monitorPoints')  http.post('/login')  http.post('/analysis/image')  │
└────────────────────────────┬────────────────────────────────────────┘
                             │ baseURL: /api
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                    Vite Proxy (vite.config.ts)                        │
│  /api/analysis/* → Flask:5000    /api/* → json-server:3000 (去掉前缀)  │
└─────────────┬──────────────────────────────┬──────────────────────────┘
              │                              │
              ▼                              ▼
┌──────────────────────────┐    ┌──────────────────────────────────────┐
│  Flask (port 5000)       │    │  json-server (port 3000)              │
│  MOCK_RESULTS 随机返回   │    │  db.json → users, monitorPoints, alerts │
│  病害识别模拟            │    │  自定义 /login 接口                     │
└──────────────────────────┘    └──────────────────────────────────────┘
```

## 七、切换到真实后端

1. **JSON Server 替换**：修改 `vite.config.ts` 中 `/api` 的 `target` 为真实后端地址  
2. **Flask 替换**：在 `run_ai_model_prediction()` 中调用真实模型  
3. **前端内联模拟**：在 `RelatedData.vue`、`DataAnalysis.vue` 等组件中接入真实 API 或数据源
